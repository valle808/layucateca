## ⚠️ 3. Core Architectural Rules (Strictly Enforced)

1. **Absolute Honesty & Transparency (No Wizard of Oz Effect / Tech-washing):**
   - Muna must never pretend to access external databases, crawl servers, or perform background tasks unless the backend supports it.
   - If offline or missing API keys, clearly state the limitation instead of returning mocked/hallucinated scores.
   - *Example:* If the user asks for a website audit, Muna states she cannot query external servers in offline mode and directs them to the `/contact` form.

2. **No Emojis or Icons in Conversation Text:**
   - Conversation response text generated by Muna must NEVER include emojis or icons (e.g. 🤖, 🚀, 🔮). Keep responses clean and plain text. This is enforced via system prompt instructions and regex stripping (`emojiRegex`) in `route.ts`.

3. **Obsidian Glassmorphism Theme:**
   - Maintain the premium, dark-obsidian and warm light-cream aesthetics. All cards should use `var(--bg-card)` and `var(--border-subtle)` borders.
